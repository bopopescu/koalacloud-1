KOALA unterstützt eine reine S3-Darstellung eine Komfort-Darstellung. Die reine Darstellung gibt den Inhalt der Buckets exakt wieder. Da S3 ein flacher Namensraum ist, existieren in S3 keine Verzeichnisse, sondern nur Keys. Die Komfort-Darstellung emuliert Verzeichnisse wie S3Fox es macht. Bestimmte Keys dienen als Platzhalter für Verzeichnisse. Diese enden auf den Namen _$folder$. Ein Key, der einem Verzeichnis zugeordnet werden soll, hat das Namensschema: verzeichnis/unterverzeichnis/dateiname

 

Um den Anforderungen von Amazon S3 und DNS zu entsprechen, müssen die Namen von Buckets den folgenden Kriterien genügen:

 

  • Der Name kann Kleinbuchstaben, Zahlen, Punkte (.) und Bindestriche (-) enthalten
  • Der Name muss mit einer Zahl oder einem Buchstaben anfangen
  • Der Name sollte zwischen 3 und 63 Zeichen lang sein
  • Der Name darf nicht wie eine IP-Adresse formatiert sein (z.B. 192.168.5.4)
  • Der Name sollte nicht mit einem Bindestrich enden
  • Der Name darf keine zwei direkt benachbarten Punkte or Bindestriche enthalten
  • Der Name darf keine Punkte und Bindestriche direkt nebeneinander enthalten (z.B. "my-.bucket.com" und "my.-bucket" sind ungültig)
